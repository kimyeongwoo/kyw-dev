# TASK 0012 — Grilling Parity Benchmark and Tuning

## Status

DONE

## Goal

Use the Task 0011 harness to measure whether `kyw-grilling` preserves or improves the pinned upstream interview behavior, tune only evidence-backed regressions, and establish a reproducible release baseline.

## Dependencies

- `0011-grilling-eval-harness`

## In Scope

- Freeze the benchmark configuration before execution: Codex version, model, reasoning effort, scenario set, repetition count, rubric version, and pass thresholds.
- Run the same scenarios and scripted replies against the pinned upstream baseline and `kyw-grilling` in isolated read-only sessions.
- Measure critical protocol compliance, decision coverage, dependency order, unnecessary fact questions, premature action, premature convergence, turns, and token usage.
- Review raw transcripts for all critical violations and statistically or qualitatively material score differences.
- Make the smallest justified changes to `kyw-grilling` or its canonical wrapper references when evidence demonstrates a regression.
- Re-run the entire bounded benchmark after any behavior change.
- Record a compact, reproducible baseline and residual limitations for future releases.

## Out of Scope

- Changing the product workflow merely to outperform the benchmark.
- Weakening the rubric or thresholds after seeing unfavorable results.
- Expanding the evaluation platform beyond grilling behavior.
- Publishing the npm package.
- Running unbounded repetitions or committing every raw transcript.

## Risks

- Model variance can make small score differences meaningless; do not overfit to a single transcript.
- The evaluator and evaluated model may share biases, so deterministic critical checks and human review of borderline cases remain necessary.
- Skill prompt changes can improve one scenario while degrading another; always re-run the full bounded suite.
- Benchmark runs may be expensive or rate-limited.
